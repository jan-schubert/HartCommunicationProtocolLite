Data input

Finaltec.HartAnalyzer

The data bytes will be input like shown in the table below. To seperate more data bytes use the ";" (semicolon) seperator between the data bytes. In the "Size" column you can see how many space the datatype need. To be save use always the cast input variant.
Possible data byte input is for command 12 "(int24)32647;25;(short)8;3.4;(int)214;"unique";(float)2" the result in single bytes display as hex values for this input is: 87, 7F, 00, 19, 08, 00, 33, 2E, 34, D6, 00, 00, 00, 22, 75, 6E, 69, 71, 75, 65, 22, 00, 00, 00.

Attention: Without casting to a explicit type the program will user the type "Byte".
Exclusion: String and packed ascii, this types works with "" and '' like in the table below.

NOTE: Packed Ascii is currently not implementet.
